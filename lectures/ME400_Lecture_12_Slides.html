Lecture 12 - Basics of Functions¶

With some simple programs under our belts, it is time to modularize our programs by using functions. In this lecture and Lecture 13, you will learn how to define your own functions to meet a variety of needs.

Objectives¶

By the end of this lesson, you should be able to

  • Define a function that accepts (zero or more) input arguments and returns (zero or more) values.
  • Explain the meaning of a named and default argument.
  • Use unpacking to define multiple variable in a single statement.
  • Document functions using appropriate “docstrings.”
  • Include functions in flowcharts.

What is a Function?¶

A function is a block of code that can be independently executed (or “called”) that is usually given input and usually produces output. Basic structure:

def function_name(arg0, arg1, ...):  # 0 or more arguments
    """ A docstring, which is a string that
        explains what this function does.
    """
    statement0                       # 0 or more lines of code that do something, but hard to
    statement1                       #   justify a function without at least one!
    # ...
    return rval0, rval1, ...         # 0 or more return values.  Skip return if no values to be returned.

Example: Write a function named factorial that computes and returns the factorial of an integer \(n\), i.e., \(n!\).

Example: Write a function named dropped that accepts a sequence (e.g., a list or ndarray) of numbers a and an integer n. The function should return a list having the same elements as a but with the n lowest elements removed. The original order of the elements should be the same. An example use would be

In  [1]: dropped([5, 6, 4, 5, 3, 6], 2)
Out [1]: [5, 6, 5, 6]

Don’t forget a return value (if one is expected)!

In [1]:

def buggy_sum_function(x):
    s = 0
    for i in range(len(x)):
        s += x[i]

In [2]:

buggy_sum_function([1, 2]) + buggy_sum_function([3, 4]) # 10?

In some applications, programs can be simplified by using functions with multiple return values.

Example: Write a function that returns the mean and median of an array of numbers a. Example use case:

In  [1]: mean, median = mean_and_median([3, 4, 5, 10])

In  [2]: mean, median
Out [2]: (5.5, 4.5)

This multiple assignment is called unpacking. See the reading for examples.

Functions with Multiple Arguments¶

There is no limit to the number of arguments a Python function can accept.

Arguments can be made optional by providing default values.

Example: Write a function adjust_time that accepts any number of seconds, minutes, and hours and returns the number of seconds, minutes, and hours adjusted so that m < 60 and s < 60 as a formatted string like 12:34:16 for 12 hours, 34 minutes, and 16 seconds.

In [3]:

def adjust_time(s, m, h) :

    return '{:02d}:{:02d}:{:02d}'.format(h, m, s)

As defined, one must pass three values to adjust_time for s, m, and h:

In [4]:

adjust_time(67, 55, 100)

Out[4]:

'100:55:67'

Example Modify adjust_time to provide a default value of zero for m and h.
